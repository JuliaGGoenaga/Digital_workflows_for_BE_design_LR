Digital workflows for informed decision making in building envelope design: A systematic literature review.

Task classification process

An interesting pattern has been recognised across the papers that determines the classification process developed in this review. The different optimization operations held in a project are normally destined to obtain an optimal design solution. Usually, this approach entails repeating the optimization process or enriching it with new design perspectives. This is sometimes called an optimization loop [20] or feedback [36]. These definitions can be oriented to a singular optimization problem or a complete design process. To this extent, it can be stated that from the white paper to generating a design solution, several design tasks might be held to reach the optimal solution, as illustrated in the figure.

This structure is well demonstrated in some of the studied cases. Any time the variables or performance objectives are changed for a new problem-solving process a task is defined. Even when variables or performance objectives remain the same, tasks can be differentiated due to the sequential process of decision-making implementation. This is when it is clearly stated that one step was taken after the other.

A methodological framework has been developed identifying and organizing the key processes involved in multi-objective design workflows. The framework is structured around four core phases: Conceptualization, Data Preparation, Performance Analysis, and Solution Mapping.

The process results in a comprehensive map or matrix that outlines the potential tools applicable within digital workflows for the BE design process. As shown in Figure 12, these phases constitute a modular and flexible framework in which different components can be combined depending on the design goals and computational resources.
